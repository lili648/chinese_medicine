### 项目目标

基于 NLP + 知识图谱技术，对预设医学文献数据集（PubMed 500篇 + 中文 200篇）进行结构化解析，自动抽取疾病、药物、症状三类医学实体，构建 Neo4j 知识图谱，并通过 Streamlit 搭建 Web 界面，提供文献检索和交互式图谱可视化。

### 3.3 数据模型（核心设计输入）

#### Neo4j 图模型 Schema

**节点类型：**

| 标签 | 说明 | 关键属性 |
|------|------|----------|
| Article | 文献 | article_id, pmid, title, abstract, authors, journal, pub_year, language |
| Disease | 疾病实体 | entity_id, name, source |
| Drug | 药物实体 | entity_id, name, source |
| Symptom | 症状实体 | entity_id, name, source |

**关系类型：**

| 关系 | 方向 | 说明 | 属性 |
|------|------|------|------|
| MENTIONS | Article → Entity | 文献提及实体 | frequency |
| CO_OCCURS | Entity ↔ Entity | 实体共现 | frequency, confidence |
| TREATS | Drug → Disease | 药物治疗疾病 | frequency |
| HAS_SYMPTOM | Disease → Symptom | 疾病有症状 | frequency |

#### 数据量预估

| 类型 | 预估数量 |
|------|----------|
| Article 节点 | ~700 |
| Disease 节点 | ~500 |
| Drug 节点 | ~300 |
| Symptom 节点 | ~200 |
| **图谱总节点** | **~1,700** |
| MENTIONS 关系 | ~2,500 |
| CO_OCCURS 关系 | ~1,500 |
| **图谱总关系** | **~5,000** |

#### E-R 关系图

```
  ┌──────────┐         ┌──────────┐         ┌──────────┐
  │ Disease  │◄────────┤   Drug   │         │ Article  │
  │  疾病    │  TREATS │  药物    │         │  文献    │
  └────┬─────┘         └────┬─────┘         └────┬─────┘
       │                    │                    │
       │ HAS_SYMPTOM        │                    │ MENTIONS
       │                    │                    │
  ┌────▼─────┐         ┌────▼─────┐         ┌────▼─────┐
  │ Symptom  │         │   Any    │         │ Disease  │
  │  症状    │         │ Entity   │         │  Drug    │
  └──────────┘         └──────────┘         │ Symptom  │
                                            └──────────┘
       ┌──────────────────────────────────────────┐
       │        CO_OCCURS (Any Entity ↔ Any)       │
       └──────────────────────────────────────────┘
```
